## Project Identity

- **Name:** AI Agent Superpowers  
- **Publisher:** aliabbaschadhar  
- **Version:** 1.1.0  
- **Stack:** TypeScript 5.x, VS Code Extension API (^1.85), esbuild, Bun  
- **Repo:** https://github.com/aliabbaschadhar/agent-superpowers  

## Tech Stack Constraints

- Language: **TypeScript** only in `src/`  
- Bundler: **esbuild** via `esbuild.config.js`  
- Runtime: **Bun** for scripts (`scripts/`, `prebuild.js`)  
- Test runner: `@vscode/test-electron`  
- Fuzzy search: **Fuse.js** (the only runtime dependency)  

## Repository Layout

```
src/
  extension.ts          — activate() entry point
  commands/             — one file per registered VS Code command
  skills/               — SkillsManager, SkillsRepository, RemoteSync, FuzzySearch, WorkspaceScanner
  installers/           — one installer class per target agent
  tree/                 — SkillsTreeProvider + tree node helpers
  constants.ts          — shared string constants
  editorDetector.ts     — detects running editor (Cursor vs VS Code)
  logger.ts             — VS Code OutputChannel wrapper
  favoriteSkills.ts     — GlobalState persistence for favorites
  recentSkills.ts       — GlobalState persistence for recents
assets/
  skills_index.json     — bundled skills manifest (generated by prebuild.js)
  skills/               — bundled SKILL.md files (~940 skills)
  manifest.json         — category + metadata manifest
```

## Core Abstractions

| Class/Module | Responsibility |
|---|---|
| `SkillsManager` | Orchestrates skill discovery, state, queries, and remote sync |
| `SkillsRepository` | File I/O — reads bundled index + local skill sources |
| `RemoteSync` | Fetches latest skills_index from GitHub and caches it |
| `InstallationDetector` | Scans agent-specific paths to determine installed skills |
| `SkillsTreeProvider` | VS Code TreeDataProvider — drives the Sidebar tree |
| `WorkspaceScanner` | Detects technologies in workspace to recommend skills |
| `FuzzySearch` | Fuse.js wrapper for skill search |
| `*Installer` | Agent-specific file writers (Claude, Gemini, Cursor, Copilot, Generic) |

## Behavioral Rules

1. Preserve all existing disposable registrations in `context.subscriptions`.  
2. All new commands must be registered in both `extension.ts` and `package.json` `contributes.commands`.  
3. Use `log()` from `src/logger.ts` — never `console.log`.  
4. Bundled assets under `assets/` are read-only at runtime; generated by `scripts/prebuild.js`.  
5. Follow the Conventional Commits standard for commit messages.  
6. Reference `.agent/AGENT_RULES.md` for full code style and testing constraints.
